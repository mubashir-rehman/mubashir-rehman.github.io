# Using AI-Generated / Automation Content — key points

Source: https://developers.google.com/search/docs/fundamentals/using-gen-ai-content
Fetched: 2026-07-12

## Is AI content allowed?
Yes — Google rewards **quality, not production method**. AI is not penalized per
se; low-quality outcomes are.

## Acceptable vs. spam
- **Spam:** using AI to "generate many pages without adding value" →
  **scaled content abuse** policy violation (see Quality Rater guidelines 4.6.5 /
  4.6.6 on scaled + low-effort unoriginal content).
- **OK:** content that shows originality, effort, and user value.

## Disclosure / transparency
"Consider adding information on how your content was created in a way that makes
sense for your audience" — i.e. disclose substantial automation, like you'd
disclose human authorship.

## Do
- Focus on accuracy, quality, relevance.
- Validate structured data.
- Add image metadata for AI-generated visuals (IPTC for ecommerce; label
  AI-generated product data).
- Give users context about creation method.

## Don't
- Generate scaled content without added value.
- Neglect metadata (`<title>`, descriptions, alt text).
- Skip feature-specific policy compliance.

## Relevance to this site
Portfolio content is human-authored personal work → compliant. Only caveat:
if any section becomes substantially AI-assisted, add a light disclosure.
