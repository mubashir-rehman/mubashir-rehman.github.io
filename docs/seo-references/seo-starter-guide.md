# SEO Starter Guide — key points

Source: https://developers.google.com/search/docs/fundamentals/seo-starter-guide
Fetched: 2026-07-12

## Titles
Unique per page, clear and concise, accurately describe the page. Include
site/business name; page-specific value. Google pulls from `<title>` + headings.

## Meta descriptions
Short (1–2 sentences), unique per page, most relevant points. Used to generate
snippets when page content isn't a better fit.

## Headings
Use semantic order (helps screen readers). Google does **not** penalize
out-of-order headings. No ideal count — "if you think it's too much, it probably is."

## Links
- Internal: link relevant pages of your site; use **descriptive anchor text**.
- External: link only resources you trust; use `nofollow`/`ugc` for untrusted or
  user-generated links.

## Images & alt text
Place high-quality images near relevant text. Write good `alt` text describing the
image↔content relationship.

## Site structure & URLs
Descriptive URLs with real words; group topically-similar pages in directories.

## Content quality
Easy-to-read, well-organized, unique, up-to-date, helpful & reliable; cite
expert/experienced sources; don't copy others' work.

## robots.txt / noindex
Use `robots.txt` and `noindex` to opt URLs out of crawling/indexing.

## Sitemaps
Not required; list URLs you care about. Some CMSs submit automatically.

## Structured data
Valid structured data → eligibility for rich results (review stars, carousels…).

## JavaScript rendering
Google should "see the page the same way an average user does." If important
CSS/JS is hidden, Google "might not be able to understand your pages." Verify with
the URL Inspection Tool in Search Console.

## What NOT to do
- Keywords meta tag: unused by Google.
- Keyword stuffing: spam policy violation.
- Keywords in domain: negligible effect.
- No magic word count.
- **E-E-A-T is NOT a ranking factor** ("No, it's not").
- Duplicate content under multiple URLs is fine (copying *others'* content is not).
